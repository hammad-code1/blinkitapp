# Supabase Database Schema

To enable **LIVE MODE** in Minute Metrics, please run the following SQL commands in your Supabase SQL Editor.

## 1. Profiles Table
Stores user roles and information.
```sql
create table public.profiles (
  id uuid references auth.users on delete cascade not null primary key,
  email text,
  role text default 'user',
  updated_at timestamp with time zone default timezone('utc'::text, now())
);

-- Enable Realtime
alter publication supabase_realtime add table public.profiles;
```

## 2. Products Table
```sql
create table public.products (
  product_id text primary key,
  product_name text not null,
  category text,
  price decimal(10,2),
  stock int default 0,
  mrp decimal(10,2),
  updated_at timestamp with time zone default timezone('utc'::text, now())
);

-- Enable Realtime
alter publication supabase_realtime add table public.products;
```

## 3. Orders Table
```sql
-- Drop the old table if it exists to reset the PK constraint
drop table if exists public.orders;

create table public.orders (
  id bigint generated by default as identity primary key,
  order_id text not null,
  user_id text,
  product_id text references public.products(product_id),
  quantity int default 1,
  revenue decimal(10,2),
  order_date timestamp with time zone default timezone('utc'::text, now()),
  city text,
  delivery_time_mins int,
  item_rating int,
  metadata jsonb default '{}'::jsonb
);

-- Enable Realtime
alter publication supabase_realtime add table public.orders;
```

## 4. Master Data (Admin Only / Fallback)
```sql
create table public.master_data (
  id bigint generated by default as identity primary key,
  role text,
  data jsonb,
  created_at timestamp with time zone default timezone('utc'::text, now())
);
```

## Security Rules (RLS)
For local development, you can disable RLS for these tables or set up simple policies allowing authenticated users to read/write.

```sql
alter table public.profiles enable row level security;
create policy "Allow internal read" on public.profiles for select using (true);
create policy "Allow self update" on public.profiles for update using (auth.uid() = id);

alter table public.products enable row level security;
create policy "Public read" on public.products for select using (true);
create policy "Auth write" on public.products for insert with check (auth.role() = 'authenticated');

alter table public.orders enable row level security;
create policy "Public read" on public.orders for select using (true);
create policy "Auth write" on public.orders for insert with check (auth.role() = 'authenticated');
```
